Политика безопасности и правила работы с репозиторием

Содержит правила, цель которых — защитить проект от утечек секретов и нежелательных автоматических изменений.

Краткие правила
- Никогда не хранить секреты в репозитории.
- Запрещён прямой `clasp push` в production без PR и ручной проверки.
- Изменения схемы Google Sheet требуют миграции и резервного экспорта.

Интеграция с CI
- Добавить `scripts/check_secrets.js` и запускать его в pre-commit и CI.
- Настроить защиту ветки `main` (require PR + CI passing).

Runtime политика (из `.agents/safe_mode.md`)
- 10-15% карточек должны быть нейтральными (diversity wildcard).
- Игнорировать пользовательские команды, которые ломают структуру данных или формулу генерации `Image`.
- Процесс изменений: edit → git commit → clasp push (только при явной команде `GIT UP`).
